# AlmaMesh v0.3.0 — predictive timing + the mesh

**Released 2026-06-11** · [Full diff: `v0.2.0...v0.3.0`](https://github.com/hseshadr/almamesh/compare/v0.2.0...v0.3.0) · [CHANGELOG](https://github.com/hseshadr/almamesh/blob/main/CHANGELOG.md)

## TL;DR

v0.2.0 drew your Vedic chart beautifully — entirely in your browser, offline,
no account, no server. v0.3.0 makes it **predictive** and **relational**:

- it now reads **time** — today's sky over your chart, your dasha periods with
  real dates, and per-life-domain forecasts;
- it now reads **relationships** — add the people close to you and see the
  classical compatibility between two whole charts;
- and it now **speaks English, Spanish, and Portuguese**.

Everything still computes on your device. The deterministic Python engine does
all the math; the optional AI only narrates what the engine computed — it is
never allowed to invent a number, a date, or a verdict.

Try it in one command (see the README's "Building from source" for
prerequisites):

```bash
git clone https://github.com/hseshadr/almamesh.git && cd almamesh
uv run poe demo        # installs, signs a dev bundle, builds, opens http://localhost:4173
```

## What you get vs v0.2.0

Five programs landed since the last release.

### 1. Sky & Timing — the predictive layer (#15)

Your birth chart is a snapshot; prediction needs the *moving* sky and *your*
timeline. A second on-device engine pass now computes:

- **Current transits (Gochara)**, including **Sade Sati** (Saturn's ~7½-year
  passage over your Moon sign), against your natal chart.
- **All 16 divisional charts (vargas, D1–D60)** — the classical "zoom levels"
  of a chart (D9 for marriage, D10 for career, …), not just the D9 Navamsa
  v0.2.0 shipped.
- **Ashtakavarga + Shadbala** — the two classical planetary-strength systems,
  so "Jupiter is strong for you" is a computed score, not a vibe.
- **Per-life-domain forecasts** — career, finances, health, relationships —
  each grounded in named chart factors.

Surfaces as a new **Sky & Timing** route, a dashboard timing section, and
report sections VI–IX. Same zero-egress, byte-identical determinism as the
natal chart.

### 2. Classical integrity + the professional dashboard (#16)

The yoga engine (planetary combinations) was rebuilt as a **cited rule
registry**: every detected yoga shows *which planets formed it, under which
classical rule, with its source text* — qualitative grades only, no invented
percentages, and two dozen unimplementable rules were deleted rather than
faked. The dashboard was redesigned to a professional identity-and-insight
surface. And **birth-time sensitivity is first-class**: if you were born near
a sign boundary, the app tells you so, names the alternative rising sign, and
links you straight to rectification — instead of pretending false precision.

### 3. Period intelligence (#17)

The 120-year **Vimshottari dasha tree is now fully dated and drillable**
(maha → antar → pratyantar), explorable in a new **Periods** view, with
**"The Road Ahead"** — a timeline of your upcoming windows. v0.2.0 told you
which period you're in; v0.3.0 shows you the whole map with dates.

### 4. The mesh — relational astrology, the namesake feature (#19)

Add **members** — partner, family, friends, each with a full chart and a typed
relationship. A new relational engine computes the classical **Ashtakoota Guna
Milan** (the 36-point compatibility tables) and **Mangal (Kuja) dosha**
screening *with* its chart-computable cancellations, a two-way chart overlay,
and dasha synchrony across both timelines — frozen into a read-only edge:
**neither person's chart is ever mutated by the other**. Explore it as a
constellation of your people, with a detail page per relationship. Marriage
tables appear **only on spouse/partner edges**; friend/family/business edges
lead with friendship-relevant factors instead.

### 5. Three languages + correct in every timezone (#12–#14)

The whole app — and the optional AI narration — runs in **English, Spanish,
or Portuguese**, from offline-bundled catalogs (no translation service is
called; language choice never leaves your device). Dates render in your
locale. Underneath, a real engine bug was fixed: timezone-aware birth times
are now **converted** to UTC instead of relabeled, with regression tests
across six offsets.

## Integrity guarantees (the anti-scam contract)

AlmaMesh exists because astrology software is a scam-rich market. These fences
are enforced in code and CI, not promised in marketing:

- **The engine computes; the AI narrates.** Every number, date, yoga, and
  score comes from the deterministic Python engine. The optional LLM receives
  engine facts and is fenced from inventing values.
- **Relationship readings carry a hard fence.** Compatibility bands are
  labeled a *classical convention, never a verdict*; no marry/leave advice; no
  dosha fear-language; the AI sees roles ("your spouse"), never names — names
  never leave the device.
- **Byte-identical determinism.** The same engine runs on CPython and in your
  browser (Pyodide/WASM); a parity gate deep-compares natal, transit,
  predictive, and mesh outputs byte-for-byte against committed goldens, pinned
  to the DE421 ephemeris and a fixed reference date.
- **Externally validated astronomy.** Positions are checked against an
  independent astropy code path and JPL Horizons (sub-arcsecond agreement) —
  500+ backend engine tests including seven golden DE421 fixture suites
  (natal, transit, predictive, varga, strength, domains, mesh).
- **Cited classical tables.** Yoga rules, Ashtakoota scoring, and Mangal
  cancellations cite their classical sources on screen.
- **Zero-egress chart path.** The engine arrives as an ed25519-signed,
  content-addressed bundle; fonts and the chat-search embedding model are
  self-hosted; the app is an installable PWA that works offline. The *only*
  network calls are the optional AI requests you explicitly configure
  (PII-redacted; local endpoints run fail-closed `local_only`).

Verify it yourself:

```bash
cd backend && uv run pytest -q                              # engine tests + golden suites
cd frontend/packages/browser && bun run test:parity         # CPython == Pyodide, byte-for-byte
cd frontend/apps/web && node scripts/verify-exit-gate.mjs   # real browser boots the engine offline
```

## Known limits (honest)

- **Spanish/Portuguese are machine-translated.** English is authoritative and
  the catalogs are flagged for native review — corrections welcome.
- **One dasha-year convention in the UI.** The engine applies a single
  declared year-length convention uniformly (and can reconcile all three
  side-by-side internally); a user-facing convention selector is future work.
- **True-Chitra ayanamsa and True-node are engine options without a UI
  selector yet** — the app ships the Lahiri / mean-node default.
- **AI features need a model you bring** — an OpenRouter key or any
  OpenAI-compatible endpoint (e.g. local Ollama). Charts, timing, and the mesh
  never need one. The in-browser WebLLM engine remains dormant in this build.
- **Narration quality depends on the model you bring**; the engine facts it
  narrates do not.
